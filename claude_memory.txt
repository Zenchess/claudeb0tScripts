## Session (2026-01-30 14:14): Wayland Input SOLVED
- **BREAKTHROUGH:** Found working input method for Wayland/Hyprland!
- ydotool (Wayland-compatible input tool) + hyprctl (window focus) combination works
- Command pattern: `hyprctl dispatch focuswindow address:<window_addr> && sleep 0.3 && ydotool type '<command>' && ydotool key 28:1 28:0`
- Window address for hackmud: `hyprctl clients -j | jq -r '.[] | select(.class | test("hackmud"; "i")) | .address'`
- Key 28 = Enter key (28:1 = press, 28:0 = release)
