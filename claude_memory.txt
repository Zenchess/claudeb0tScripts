## SCRIPTING TIPS
- **CRITICAL**: Never store #db macro in a variable! Use #db.f(), #db.i(), #db.us() directly. Storing it (var d = #db) causes PARSE ERROR.
- **SPACES, NEWLINES, and SINGLE-LINE COMMENTS DON'T COUNT** toward character limit!
  - Spaces: free, use for formatting
  - Newlines: free, use for readability
  - Single-line comments `// ...`: free, use for documentation
  - Block comments `/* */`: DO count (avoided by regex preprocessing)
- Use #up scriptname delete to delete a script
- Use sys.cull{i:[index]} with kernel.level_bypass to trash unwanted upgrades
- psi_nabby.lib has RNG functions: #fs.psi_nabby.lib().generationFunctions.rng(1,10)
- _RAND(n) global does Math.floor(Math.random() * n), range [0,n)
- Math.random() is hackable in hackmud - be careful with RNG-dependent scripts

### Memory Scanner Issue - DIAGNOSED

The mem_scanner is producing garbled output for command responses. Investigation findings:

1. **Chat/JSON works fine**: The JSON scanner successfully captures chat messages from the hackmud web API responses stored in memory. Chat API (chat_api.py) works because it uses official API.

2. **Backtick scanner produces garbage**: The "backtick_colored" pattern (`[A-Za-z0-9]`) matches random binary data like sequential test patterns (`4d4e4f4f4g...`), not actual game content.

3. **Command output not stored as plaintext**: The scripts.fullsec output (list of scripts) isn't stored as readable text in memory. The >> prompts are followed by binary glyph data (likely Unity TextMeshPro indices), not ASCII.

4. **Unity color tags exist but sparse**: Real game content uses `<color=#XXXXXXXX>text</color>` format, found in memory but not for command output.

**Root cause**: The game likely stores terminal display as glyph indices or uses a rendering system that doesn't store plaintext. The old memory patterns no longer work.

**Better approach discovered (from loki repo - github.com/AkiraYasha/loki)**:
1. Uses Mono runtime introspection to find vtable pointers in memory
2. Scans for managed objects (Window, Shell, Parser classes) by matching vtables
3. Reads from a circular queue buffer with `array`, `head`, `tail`, `size` fields
4. Text has Unity color tags `<color=#...>` that need stripping
5. Special char encoding: È → <, É → >

This approach requires:
- ELF symbol parsing to find `mono_get_root_domain`
- Traversing assembly/class hierarchy from root domain
- Scanning memory regions for vtable matches
- Following object references to output queue
