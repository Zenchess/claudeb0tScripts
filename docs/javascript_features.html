Hackmud JavaScript Feature Reference

Comprehensive testing results for JavaScript and WebAssembly support in hackmud

TL;DR: Hackmud runs approximately ES2019-level JavaScript with an older Esprima-based parser (~ES6 syntax). Modern runtime methods are polyfilled, but modern syntax (optional chaining, nullish coalescing, object spread) causes parse errors.

Quick Feature List

✅ Fully Supported

  • var, let, const declarations
  • Arrow functions: (x) => x * 2
  • Template literals: `Hello ${name}`
  • Destructuring: let {a, b} = obj, let [x, y] = arr
  • Array spread: [...arr, 4, 5]
  • Rest parameters: function(...args)
  • Default parameters: function(x = 10)
  • Classes: class Foo extends Bar {}
  • for...of loops
  • Map, Set, Symbol
  • All Array methods: map, filter, reduce, flat, includes, etc.
  • Object.keys/values/entries/fromEntries
  • JSON.stringify/parse
  • BigInt
  • String.padStart/padEnd/replaceAll
  • WebAssembly (Module, Instance, Memory, compile)

❌ Not Supported

  • async / await — PARSE ERROR
  • Promise — runtime error
  • Optional chaining: obj?.prop — PARSE ERROR
  • Nullish coalescing: a ?? b — PARSE ERROR
  • Object spread: {...obj} — PARSE ERROR
  • Exponentiation operator: 2 ** 3 — PARSE ERROR
  • Generators: function* — PARSE ERROR
  • globalThis — undefined
  • Array.at(), findLast, toSorted

ES2020+ Syntax (Not Supported)

⚠️ These features cause PARSE ERRORS:

The hackmud parser does not understand this syntax. Scripts using these will fail to upload.

Feature Status Workaround
?. optional chaining PARSE ERROR obj && obj.prop
?? nullish coalescing PARSE ERROR (x !== null && x !== undefined) ? x : default
{...obj} object spread PARSE ERROR Object.assign({}, obj, {newProp: val})
** exponentiation PARSE ERROR Math.pow(base, exp)

WASM Example: Add Function

// WASM bytes for: (func (export "add") (param i32 i32) (result i32) local.get 0 local.get 1 i32.add)
const wasmBytes = new Uint8Array([
  0x00, 0x61, 0x73, 0x6d, 0x01, 0x00, 0x00, 0x00,
  0x01, 0x07, 0x01, 0x60, 0x02, 0x7f, 0x7f, 0x01, 0x7f,
  0x03, 0x02, 0x01, 0x00,
  0x07, 0x07, 0x01, 0x03, 0x61, 0x64, 0x64, 0x00, 0x00,
  0x0a, 0x09, 0x01, 0x07, 0x00, 0x20, 0x00, 0x20, 0x01, 0x6a, 0x0b
]);

const module = new WebAssembly.Module(wasmBytes);
const instance = new WebAssembly.Instance(module, {});
const result = instance.exports.add(3, 5);  // Returns 8

Best Practices

Writing Hackmud Scripts

  • Use let and const instead of var
  • Arrow functions work great for callbacks
  • Destructuring is fully supported - use it!
  • Avoid async/await - use callbacks or synchronous code
  • Use Math.pow() instead of **
  • Use Object.assign() instead of object spread
  • For optional property access, use obj && obj.prop

Script Template

function(context, args) {
    // ✅ ES6 features work
    const { caller } = context;
    let result = [];
    
    // ✅ Arrow functions
    const double = (x) => x * 2;
    
    // ✅ Template literals
    const msg = `Hello ${caller}`;
    
    // ✅ Destructuring
    const { param1, param2 = "default" } = args || {};
    
    // ✅ Modern array methods
    const filtered = [1, 2, 3, 4, 5]
        .filter(x => x > 2)
        .map(x => x * 2);
    
    // ❌ DON'T use these:
    // async/await
    // obj?.prop
    // a ?? b
    // {...obj}
    // 2 ** 3
    
    return { ok: true, data: filtered };
}
